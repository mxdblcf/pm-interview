# HTTP 协议与网络基础

## HTTP 请求方法

| 方法 | 描述 | 幂等 | 安全 |
|------|------|------|------|
| GET | 获取资源 | ✅ | ✅ |
| POST | 创建资源 | ❌ | ❌ |
| PUT | 替换资源 | ✅ | ❌ |
| PATCH | 部分更新 | ❌ | ❌ |
| DELETE | 删除资源 | ✅ | ❌ |

## HTTP 状态码

### 分类总览

```
1xx — 信息性状态码（请求已接收，继续处理）
2xx — 成功状态码（请求已成功处理）
3xx — 重定向状态码（需要进一步操作）
4xx — 客户端错误（请求有语法错误或无法完成）
5xx — 服务器错误（服务器处理请求出错）
```

### 高频状态码

- **200 OK** — 请求成功
- **201 Created** — 资源创建成功
- **204 No Content** — 成功但无返回内容
- **301 Moved Permanently** — 永久重定向
- **302 Found** — 临时重定向
- **304 Not Modified** — 缓存可用
- **400 Bad Request** — 请求语法错误
- **401 Unauthorized** — 未认证
- **403 Forbidden** — 已认证但无权限
- **404 Not Found** — 资源不存在
- **429 Too Many Requests** — 请求频率过高
- **500 Internal Server Error** — 服务器内部错误
- **502 Bad Gateway** — 网关错误
- **503 Service Unavailable** — 服务不可用

## TCP 三次握手

```
Client                              Server
  │                                    │
  │──── SYN (seq=x) ────────────────→ │
  │                                    │
  │←─── SYN-ACK (seq=y, ack=x+1) ──  │
  │                                    │
  │──── ACK (ack=y+1) ──────────────→ │
  │                                    │
  │←──── 连接建立 ────→               │
```

### 为什么是三次而不是两次？

防止**已失效的连接请求报文段**突然又到达服务器，产生错误。如果只有两次握手，服务器可能会为一个已经过期的请求建立连接，浪费资源。

## 面试常问

### Q: GET 和 POST 的区别？

| 特性 | GET | POST |
|------|-----|------|
| 数据位置 | URL Query String | Request Body |
| 数据长度 | 受 URL 长度限制 | 无限制 |
| 缓存 | 可被缓存 | 不可缓存 |
| 安全性 | 参数暴露在 URL | 相对安全 |
| 幂等性 | 幂等 | 非幂等 |
| 书签 | 可收藏为书签 | 不可收藏 |

### Q: Cookie、Session、Token 的区别？

- **Cookie**: 存储在客户端，每次请求自动携带，大小限制 4KB
- **Session**: 存储在服务端，通过 SessionID（Cookie 中传递）标识
- **Token (JWT)**: 无状态，服务端不存储，自包含用户信息
